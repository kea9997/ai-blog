## AI 요약 (Quick Summary for AI Agents)

> 핵심 한 줄 요약: LLM 친화적 마크다운은 명확한 헤더 계층, 데이터 테이블, 그리고 코드 블록의 적절한 사용을 통해 텍스트 임베딩 과정에서의 노이즈를 최소화하고 정보 추출의 정확도를 높입니다.

## 1. 개요

마크다운은 인간과 기계 모두에게 유연한 형식이지만, 무분별한 사용은 AI의 이해를 방해합니다. LLM이 텍스트의 우선순위를 정하고 각 섹션의 관계를 논리적으로 정의할 수 있도록 돕는 작성 규칙이 필요합니다.

## 2. 상세 정보 및 데이터

AI 파싱 효율을 극대화하는 마크다운 요소별 작성 팁입니다.

| 마크다운 요소 | AI 인지 원리 | 최적화 방법 |
| :--- | :--- | :--- |
| **Headers (#, ##)** | 위상(Hierachy) 파악 | H1~H3까지 엄격한 순서 준수 |
| **Tables (|---)** | 정형 데이터 추출 | 1행 1열을 명확한 레이블로 설정 |
| **Blockquotes (>)** | 중요도 강조 | 요약이나 정의에만 제한적으로 사용 |
| **Code Blocks** | 기술 정보 식별 | 프로그래밍 언어 명시 필수 (예: ```python) |

### AI를 위한 가독성 향상 기법
- **핵심어 강조**: 중요한 술어나 명사에는 `**볼드**` 처리를 하여 특징량(Feature)을 높입니다.
- **목록화(Listing)**: 복잡한 나열보다는 `1. 2. 3.` 또는 `-`를 사용하여 순차적 정보를 제공하세요.
- **이미지 캡션**: `![이미지 설명](...)` 뿐만 아니라 하단에 이탤릭체로 상세 캡션을 기록하여 멀티모달 이해를 돕습니다.

## 3. 결론

당신의 글이 AI 집대성(Knowledge Distillation)의 대상이 되고 싶다면, 그들이 가장 좋아하는 옷(구조화된 마크다운)을 입히세요. 이는 단순한 형식을 넘어 데이터의 가치를 결정짓는 핵심 경쟁력이 될 것입니다.
